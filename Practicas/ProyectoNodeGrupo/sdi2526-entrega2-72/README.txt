Inciso:
El proyecto está totalmente terminado, revisado y funcional, pero queríamos recalcar las siguiente situaciones que hemos sufrido la semana de la entrega del proyecto para dejar constancia de unos sucesos que nos han pasado con el IDE de IntelliJ y el uso de Maven:

- Hemos detectado que, al importar el proyecto en IntelliJ IDEA, algunos compañeros podían experimentar errores relacionados con el reconocimiento de archivos y, especialmente, con la ejecución de la pila de tests. Estos problemas estaban vinculados con la forma en que el IDE detectaba la ruta del proyecto y la base de datos, hasta el punto de que en algunos casos fue necesario ajustar manualmente la configuración desde Run -> Edit Configurations e incluir parámetros en VM options. Por ejemplo, en el caso de un compañero fue necesario añadir:
-DbaseUrl=http://localhost:3000 -DnodeProjectPath=C:\Users\Usuario\GitHub\prueba\sdi2526-entrega2-72

- Además, también tuvimos incidencias con la configuración del SDK, ya que los tests están preparados para Java 17 y en algunos equipos se estaba usando la versión 25, lo que provocó errores adicionales (no sabemos bien por qué). En ese sentido, al hacer pull desde GitHub en un momento en el que había cambios, podían surgir problemas con las dependencias, el pom.xml y la detección correcta de la estructura del proyecto, afectando al funcionamiento del IDE. También puede ser necesario hacer clic derecho sobre el pom.xml y seleccionar Add as Maven Project, ya que con esa acción los tests pueden volver a cargarse correctamente y quedar ejecutables.

- Aun así, la versión subida a GitHub incluye los tests dentro del propio repositorio, en la rama main, con el objetivo de dejar constancia de que han sido probados de forma intensiva por el grupo y de que funcionan correctamente. Por tanto, al descargarla deberían ejecutarse sin problema. Esperamos que estos errores no se repitan para el profesor y que se trate simplemente de un desajuste entre distintas configuraciones de IntelliJ, del SDK o del entorno local. En principio, la versión entregada en formato .zip debería ser totalmente funcional y no presentar estos problemas si se abre o importa de forma normal.
